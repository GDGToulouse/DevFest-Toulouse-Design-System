# Preview cards

21 small HTML pages, one per design aspect, originally generated by Claude Design (claude.ai/design). Each card is a self-contained, styleable display that can be screenshotted, embedded in a docs site, or rendered side-by-side as a quick design system gallery.

## Shared CSS

All cards `@import` [`../../tokens/colors_and_type.css`](../../tokens/colors_and_type.css) via [`_card.css`](_card.css), so they always reflect the current source-of-truth tokens. Editing a token in `colors_and_type.css` automatically updates every card.

## How to render

These are static HTML files. Open any one directly in a browser, or set up a tiny local static server from the repo root:

```bash
# From the repo root
python -m http.server 8000
# then browse to http://localhost:8000/examples/preview/sponsor-tiers.html
```

> ⚠️ When updating tokens in [`../../DESIGN.md`](../../DESIGN.md), remember to mirror the change in [`../../tokens/colors_and_type.css`](../../tokens/colors_and_type.css) — the cards read from the CSS, not the YAML.

## When to add a card

When a new design decision is locked in (new component, new color role, new typography token), add a card here so the design system has a visual proof. Match the existing card style: import `_card.css`, keep it visually self-contained, and label every value with its hex / token name.
