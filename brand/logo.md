# Logo — DevFest Toulouse

## Anatomie

Le logo se compose de **deux blocs** :

1. **Logo DevFest** officiel — les chevrons `<>` (typographie Google Sans Bold)
2. **Typogramme « TOULOUSE »** — lettres en lévitation, carrés symbolisant l'univers / le web / les étoiles / le pixel

> « Toulouse » occupe **plus d'espace** que « DevFest ». Cet équilibre n'est jamais inversé.

## Déclinaisons

Cinq variantes officielles couvrent tous les cas d'usage :

| Variante | Mode | Quand l'utiliser |
|----------|------|------------------|
| **Brique** | Light Mode | Usage principal sur fond clair |
| **Bicolore** | Light Mode | Variante sur fond clair (plus de contraste) |
| **Brique** | Dark Mode | Sur fond sombre |
| **White** | Dark Mode | Monochrome blanc, fonds photographiques sombres |
| **Minimal** | Tous fonds | Espaces contraints : favicon, avatar réseaux sociaux, header de site |

Tous les fichiers : [`../assets/logo/`](../assets/logo/) — formats SVG, PNG, JPG, EPS, PDF, en RVB et CMJN.

## Usages typiques

| Contexte | Variante |

| Header de site | Minimale (avatar 48 px) |

| Favicon | Minimale (carré) |

| Présentations slides | Bicolore Dark Mode si fond sombre, sinon Brique Light |

## Règles d'usage

✅ **À faire**
- Respecter une **zone de protection** d'au moins la hauteur du chevron `<` autour du logo
- Sur fond photographique : poser le logo sur un overlay sombre 30 % ou un conteneur blanc semi-transparent
- Conserver les proportions natives (pas de scale non uniforme)
- Privilégier les SVG pour le web, EPS/PDF pour le print

❌ **À ne jamais faire**
- Déformer, étirer, comprimer le logo
- Pivoter le logo (sauf la croix occitane décorative, voir [illustrations.md](illustrations.md))
- Modifier les couleurs hors des déclinaisons officielles
- Ajouter d'effets : ombre portée, contour, dégradé, biseau
- Utiliser le logo comme texture ou motif répété
- Remplacer la typographie « TOULOUSE » par un autre nom de ville pour un sister event — un tel événement doit avoir son propre logo

## Cas particulier : favicon et avatar

Pour les espaces très contraints (16×16 px favicon, miniatures), utiliser exclusivement la **version minimale** disponible dans [`../assets/logo/favicon/`](../assets/logo/favicon/) et [`../assets/logo/Avatar_400x400px.png`](../assets/logo/).
